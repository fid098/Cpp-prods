# Phase 3 — OOP

**C++ concepts covered:** classes, constructors, destructors, encapsulation, composition, inheritance, polymorphism, virtual functions, copy/move semantics, rule of five

Phase 3 organises your code into types. You will model a realistic financial domain — instruments, portfolios, brokerages — using object-oriented design.

## Projects

| # | Project | Portfolio story | Key concepts |
|---|---------|----------------|--------------|
| 01 | [instrument-pricing-domain](01-instrument-pricing-domain/) | Equities, bonds, options with pricing | classes, constructors, encapsulation |
| 02 | [portfolio-and-position-model](02-portfolio-and-position-model/) | Portfolio and position valuation | composition, copy constructors |
| 03 | [brokerage-simulator](03-brokerage-simulator/) | Broker accounts, orders, fills, fees | inheritance, polymorphism, virtual functions |
| 04 | [rule-of-five-market-data-feed](04-rule-of-five-market-data-feed/) | Safe resource ownership for a data feed | destructors, copy/move semantics, rule of five |
